## 1. Brain Packageの定義

### 1.1 Brain Packageとは
Brain Packageとは、1つのBrain（特定ドメインの知識・実験履歴・自己評価・学習履歴）を構成するファイル群を1ディレクトリにまとめた単位であり、Brain Bankへの保管・[active]への装着・エクスポートの基本単位となる。[file:1]

### 1.2 構成要素
各Brain Packageは、以下のファイルを必ず含む。[file:1]

- knowledge_graph.db（Neo4j dump）
- vector_index.faiss（FAISSインデックス）
- experiment_log.db（SQLite）
- learning_timeline.db（SQLite）
- self_eval.json
- subskill_manifest.json
- metadata.json

※ export 時には、これらに加えて brain_api.py が生成される。[file:1]

## 2. Brain Bankディレクトリ構造

### 2.1 ルート構造
標準のBrain Bankディレクトリ構造は次の通りとする。[file:1]

```text
C:/SBA/brain_bank/
├── _blank_template/               ← まっさらな脳みそのマスターコピー（変更禁止）
│   ├── knowledge_graph.db         ← Neo4j dump（空）
│   ├── vector_index.faiss         ← FAISSインデックス（空）
│   ├── experiment_log.db          ← SQLite（空）
│   ├── learning_timeline.db       ← SQLite（空）
│   ├── self_eval.json             ← 全SubSkillスコア初期値0.0
│   ├── subskill_manifest.json     ← SubSkill構造定義（空配列）
│   └── metadata.json              ← {domain: null, version: "0.0", level: 0}
│
├── Python開発_v1.4/               ← 育成済みBrain（保管状態）
│   ├── knowledge_graph.db
│   ├── vector_index.faiss
│   ├── experiment_log.db
│   ├── learning_timeline.db
│   ├── self_eval.json
│   ├── subskill_manifest.json
│   └── metadata.json              ← {domain: "Python開発", version: "1.4", level: 2}
│
├── 法人営業_v0.8/
│   └── （同構造）
│
└── [active]/                      ← 現在装着・育成中のBrain（シンボリックリンク or コピー）
    └── （同構造）
```
[file:1]

### 2.2 ディレクトリの役割
- `_blank_template/`:
  - まっさらなBrainのマスターコピー。
  - 変更禁止・読み取り専用。[file:1]
- `<ドメイン名>_vX.Y/`:
  - 育成済みBrainのスナップショット。
  - Brain Bankでの保管単位。[file:1]
- `[active]/`:
  - 現在装着中・育成中のBrain。
  - Hot-Swap・学習ループ・自己実験の対象。[file:1]

## 3. ファイル別仕様概要

### 3.1 knowledge_graph.db（Neo4j dump）
- 役割: Knowledge Base（Layer1）のグラフ構造（知識ノード・関係性エッジ）を表現。[file:1]
- フォーマット: Neo4j Community Edition互換のダンプファイル。
- 用途:
  - Brain内の概念関係・依存関係の表現。
  - 矛盾検出・再構造化・SubSkill間関係の分析。

### 3.2 vector_index.faiss
- 役割: Knowledge Base（Layer1）のベクトル検索インデックス。[file:1]
- フォーマット: FAISSインデックスバイナリ。
- 用途:
  - 意味的類似検索（テキストチャンクの類似度検索）。[file:1]
  - 重複検出（コサイン類似度 > 0.92 で重複判定）。[file:1]

### 3.3 experiment_log.db（SQLite）
- 役割: Experiment Log（Layer2）の永続DB。[file:1]
- 用途:
  - 自己実験の全履歴（成功・失敗・部分成功）を削除禁止で保管。[file:1]

### 3.4 learning_timeline.db（SQLite）
- 役割: Learning Timeline（Layer4）の永続DB。[file:1]
- 用途:
  - 取得日時・ソース種別・URL・ハッシュ・SubSkill・鮮度スコア・古い情報フラグなどを管理。[file:1]

### 3.5 self_eval.json
- 役割: Self-Evaluation（Layer3）のスコア・フラグ・スケジュール情報を保持。[file:1]
- 用途:
  - SubSkillごとの知識密度スコア、弱点フラグ、Lv、次回自己評価タイミングなどを管理。[file:1]

### 3.6 subskill_manifest.json
- 役割: 当該BrainのSubSkill構造定義（SubSkill一覧とその意味・関係）。[file:1]
- 用途:
  - Knowledge Base / Experiment Log / Learning Timeline / Self-Evaluationの共通軸として使用。[file:1]

### 3.7 metadata.json
- 役割: Brain全体に関するメタ情報（ドメイン名・バージョン・育成Lv・保存日時など）。[file:1]
- 用途:
  - Brain Bank一覧表示、Hot-Swap、エクスポート時の識別に使用。[file:1]

## 4. metadata.json 詳細仕様（テンプレ）

### 4.1 必須フィールド
```json
{
  "domain": "Python開発",
  "version": "1.4",
  "level": 2,
  "created_at": "2026-03-01T12:34:56Z",
  "last_saved_at": "2026-03-30T03:12:00Z",
  "description": "Python開発に関するBrain。Lv2達成。",
  "tags": ["Tech", "Python", "開発"],
  "brain_id": "uuid-ここにUUID",
  "source": "sba",
  "exported_at": null
}
```

- `domain`: ドメイン名（例: "Python開発", "法人営業"）。[file:1]
- `version`: バージョン文字列（例: "1.4"）。[file:1]
- `level`: 育成レベル（1〜3）。[file:1]
- `created_at`: Brain生成日時。
- `last_saved_at`: 最終保存日時。[file:1]
- `description`: 任意の説明文。
- `tags`: Brain分類用タグ。
- `brain_id`: 一意識別子（UUID推奨）。
- `source`: "sba" 固定（他システムが生成したBrainとの区別用）。
- `exported_at`: エクスポートした日時（未エクスポートならnull）。

### 4.2 Blank Templateのmetadata
```json
{
  "domain": null,
  "version": "0.0",
  "level": 0,
  "created_at": "2026-03-01T00:00:00Z",
  "last_saved_at": "2026-03-01T00:00:00Z",
  "description": "Blank Brain Template",
  "tags": [],
  "brain_id": "uuid-blank-template",
  "source": "sba",
  "exported_at": null
}
```

## 5. self_eval.json 概要仕様

### 5.1 構造イメージ
```json
{
  "level": 2,
  "last_eval_at": "2026-03-30T02:00:00Z",
  "next_eval_at": "2026-03-31T02:00:00Z",
  "subskills": {
    "設計": {
      "density": 0.92,
      "weak": false,
      "priority": 0.3
    },
    "デバッグ": {
      "density": 0.58,
      "weak": true,
      "priority": 0.9
    }
  }
}
```

- `level`: Brain全体の育成Lv（Lv.1〜3）。[file:1]
- `last_eval_at` / `next_eval_at`: 自己評価実行タイミング。[file:1]
- `subskills`: SubSkillごとの知識密度スコア（0.0〜1.0）、弱点フラグ（0.6以下でtrue）、学習優先度指標。[file:1]

## 8. Brainエクスポート形式

### 8.1 エクスポート出力構造
```text
C:/exports/Python開発_v1.4/
├── knowledge_graph.db         ← Neo4j dump（読み取り専用）
├── vector_index.faiss         ← FAISSインデックス（読み取り専用）
├── experiment_log.db          ← 成功/失敗体験DB（読み取り専用）
├── subskill_manifest.json     ← SubSkill構造・タグマップ
├── metadata.json              ← ドメイン・バージョン・達成Lv・エクスポート日時
└── brain_api.py               ← 実務ツール用APIラッパー（SBA依存なし・スタンドアロン動作）
```
[file:1]

### 8.2 読み取り専用ポリシー
- エクスポートされたDB/インデックスファイルは、実務ツール側では**読み取り専用**として扱う。[file:1]
- 実務ツールが書き込みを行いたい場合は、別途コピーを取り、SBA側Brainとは切り離して扱う。

### 8.3 brain_api.py の役割
- 実務ツールがエクスポート済みBrainをSBAなしで利用するためのスタンドアロンAPIクライアント。[file:1]
- 代表的なメソッド例:[file:1]
  - `BrainClient(path).query(question, subskill=None, top_k=5)`
  - `list_subskill(subskill_id)`
  - `info()`
  - `eval_scores()`

## 9. Hot-Swapとの整合

### 9.1 セーブ処理
- `[active]/` の全ファイルを `<domain>_vX.Y/` ディレクトリへコピー保存し、metadata.jsonのversion/last_saved_atを更新する。[file:1]

### 9.2 ロード処理
- `<domain>_vX.Y/` から `[active]/` へ全ファイルをコピーし、FAISS・Neo4j・SQLite・JSONをロードしてBrain状態を復元する。[file:1]

## 10. バージョニングと互換性

### 10.1 バージョン付与
- ディレクトリ名: `<ドメイン名>_v<version>`（例: `Python開発_v1.4`）。[file:1]
- metadata.jsonの`version` フィールドと一致させる。

### 10.2 互換性ポリシー
- スキーマ変更時は、`version` を上げ、マイグレーションスクリプトを準備する。
- 旧バージョンのBrain Packageを新バージョン実装で読み込む際は、自動マイグレーション or 互換レイヤを通す。

## 11. バックアップ・リストア方針

### 11.1 バックアップ単位
- Brain Bank全体（`C:/SBA/brain_bank/`）を定期バックアップ対象とする。[file:1]
- `_blank_template/` と主要Brainは別ストレージにもバックアップ推奨。[file:1]

### 11.2 リストア手順（概要）
- バックアップからディレクトリをリストアし、Neo4j/FAISS/SQLiteファイルをそのまま復元。
- 起動後に簡易整合性チェック（ファイル存在・バージョン一致）を行う。

## 12. 不変事項とカスタマイズ可能事項

### 12.1 不変事項
- Brain Packageは上記7ファイル構成を必須とする。[file:1]
- `_blank_template/` はマスターコピーであり、直接変更禁止。[file:1]
- `[active]/` には常に1つのBrain Packageのみが装着される。[file:1]

### 12.2 カスタマイズ可能事項
- ルートパス（`C:/SBA/brain_bank/`）は設定ファイルで変更可。[file:1]
- metadata.json / self_eval.json / subskill_manifest.json の詳細フィールド拡張。
- SQLiteテーブルスキーマのカラム追加。
